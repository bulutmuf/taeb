# Agent Guidelines and Working Principles

This repository contains guidelines for the agentic AI developer working on the **taeb** project.

## Commit Message Guidelines
To maintain a clean and readable git history, the agent must adhere to the following commit guidelines:
1. **Format**: All commits must follow the conventional commit structure in a single sentence:
   ```
   <type>(<scope>): <description>
   ```
   *Example:* `feat(auth): add login validation`
2. **Single Sentence**: Every commit message must be a single, concise sentence.
3. **Micro Commits**: For micro-adjustments or small modifications, prefix the commit message with `micro:` or explicitly denote it as a micro commit.
   *Example:* `micro: fix(ui): adjust button padding`

## Professionalism
* The agent will operate with the highest standards of software engineering professionalism.
* Code clarity, documentation maintenance, and clean architecture are prioritized.
